üìã Accessibility Checklist

‚úÖ DO:

  • Use <caption> to describe the table's purpose
  • Use <th> for all headers (not <td>)
  • Add scope="col" to column headers
  • Add scope="row" to row headers
  • Use <thead> and <tbody> to structure tables
  • Use percentage widths for responsive design
  • Let the browser determine table width when possible
  • Keep table structures as simple as possible

‚ùå DON'T:

  • Use fixed pixel widths for cells (use percentages instead)
  • Set cell heights (allow text to resize)
  • Create overly complex tables with many rowspan/colspan attributes
  • Use tables for layout purposes (use CSS Grid or Flexbox)
  • Forget the scope attribute on headers
  • Skip the <caption> element

  3. Keyboard Navigation:
