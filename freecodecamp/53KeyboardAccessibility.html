Web Accessibility: Keyboard Accessibility

Ensure your applications are usable by everyone, regardless of whether they use a mouse.

Why Keyboard Accessibility Matters

Many users rely on keyboards instead of mice due to physical disabilities, repetitive strain injuries, or personal preference. This includes users of screen readers. Keyboard accessibility ensures these users can navigate web applications effectively.

The Natural Tab Order

Interactive elements like links, buttons, and form fields are naturally focusable. Users navigate them using the Tab key in the order they appear in the DOM.

Controlling Focus with tabindex

The tabindex attribute adjusts which elements are focusable and their order.

tabindex="0"

Adds an element to the natural tab order. Useful for making non-interactive elements (like div or span) focusable.

<div tabindex="0">I am focusable now!</div>
Focusable Div
Link

tabindex="-1"

Removes an element from the natural tab order but allows it to receive focus programmatically (via JavaScript). Ideal for managing focus in modals or error messages.

<p tabindex="-1" id="error-message">Error: Invalid Input</p>

tabindex > 0 (Positive Values)

Warning: Using positive values (e.g., tabindex="1") is generally discouraged. It forces a custom tab order that can become confusing and hard to maintain as page content changes. 
<!-- Focuses Input 1, then Input 2, then Input 3 -->
<input tabindex="2" placeholder="Second">
<input tabindex="1" placeholder="First">
<input tabindex="3" placeholder="Third">

Keyboard Shortcuts with accesskey

The accesskey attribute defines a shortcut key to activate or focus an element.

Note: Combinations vary by OS/Browser (e.g., Alt + key on Windows, Ctrl + Option + key on Mac). 

<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>

Visual Focus Indicators

Users need to know which element is currently focused. Browsers provide a default ring, but you can customize it with CSS.

Custom Styling Example

button:focus {
  outline: 3px solid #005fcc;
  outline-offset: 2px;
}

Accessibility Rule: The focus indicator must have a contrast ratio of at least 3:1 against the background.

Avoiding Keyboard Traps

Crucial: A "keyboard trap" occurs when a user tabs into a component (like a modal or custom widget) but cannot tab out of it. Always ensure there is a clear way to exit via the keyboard (e.g., proper tab cycling or the Esc key).
