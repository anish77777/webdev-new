aria-labelledby vs. aria-describedby

Both attributes connect an element to text elsewhere on the page, but they serve different purposes for screen reader users. Think of it as the difference between the Name of something and its Description.

aria-labelledby aria-describedby
The "Name" (Label) The "Description" (Details)
Defines what user control is. It acts as the accessible name. Provides additional information. It acts as the accessible description.
Essential. If an element lacks a label, the user won't know what it's for. Optional. Used for hints, error messages, or extra instructions.
Overrides other labels (like <label> or inner text). Supplements the label. It is read after the label and role.
Example: "Search" Example: "Press Enter to search"
Case Study 1: aria-labelledby

When to use it:

Use it when an element doesn't have visible text that describes it, or when logical label text exists elsewhere on the page.

Scenario: A "Read More" link that needs to correspond to a specific article title.

<h2 id="article1">Accessibility in 2024</h2> <p>New standards are emerging...</p> <a href="..." aria-labelledby="article1">Read More</a>

Screen Reader Output: "Link, Accessibility in 2024"

(Without it, the user would just hear "Link, Read More", which is ambiguous.)

Case Study 2: aria-describedby

When to use it:

Use it when the element already has a name, but needs extra context, instructions, or error messages.

Scenario: A password field that requires specific characters.

<label for="pw">Password</label> <input type="password" id="pw" aria-describedby="pw-hint"> <span id="pw-hint">Must be at least 8 chars.</span>

Screen Reader Output: "Password, Secure Edit text. Must be at least 8 chars."

(The screen reader announces the Label first, then the Role, and finally the Description.)

Can you use both?

Yes! It is common to see both on complex widgets.

<!-- A complex date picker input --> <label id="date-label">Departure Date</label> <span id="date-format">MM/DD/YYYY</span> <input type="text" aria-labelledby="date-label" aria-describedby="date-format">

In this case:

  • Name (Label): "Departure Date" (Essential identification)
  • Description: "MM/DD/YYYY" (Helpful format hint)
