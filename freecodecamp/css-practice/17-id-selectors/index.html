ID Selectors

Singular, powerful, and precise targeting for unique elements.

What are ID Selectors?

ID selectors are defined by a hash symbol (#) followed by the ID name. They are used to target a single, unique element within an HTML document.

CSS
#unique {
  color: purple;
}

I am an element with the "unique" ID. I am colored purple!

I am just a normal paragraph without an ID.

Uniqueness Requirement

Unlike class selectors, an ID should be unique within an HTML document. No two elements should share the same ID. This makes them ideal for structural elements like #header, #footer, or specific unique UI components.

Specificity: (0, 1, 0, 0)

SCORE: 0.1.0.0

ID selectors have a very high specificity. They reside in the second column of the (a, b, c, d) scoring system. This means they will override any number of class or type selectors.

While powerful, ID selectors are less reusable than classes and can lead to specificity "arms races" if overused for general styling.

Summary

• Use IDs for elements that appear only once per page.

• IDs use the # prefix in CSS.

• High specificity (0, 1, 0, 0) makes them stronger than classes and tags.

• Prefer classes for reusable patterns to maintain flexible CSS.
