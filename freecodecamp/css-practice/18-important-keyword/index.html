The !important Keyword

The ultimate authority in the CSS cascade.

What is !important?

The !important keyword is used to give a style rule the highest priority, allowing it to override any other declarations for a property, no matter how specific the other selectors are.

CSS Syntax
.para {
  background-color: black !important;
  color: white !important;
}

Overriding Inline Styles

Normally, inline styles have the highest precedence. However, !important can even bypass these direct style attributes.

This paragraph has inline styles: background-color: lightblue; color: black;.
But thanks to !important in the CSS file, it's actually black and white!

Best Practices

While powerful, overusing !important can make your CSS fragile and difficult to debug. Here are the golden rules:

  • Use sparingly: Only use it when you've exhausted specificity options.
  • Third-party overrides: Useful for overriding styles from libraries you don't control.
  • Avoid for layout: Don't use it to "fix" broad layout issues.
  • Maintenance debt: Each !important makes future changes harder.

Summary

• !important forces a style to win, regardless of specificity.

• It is placed after the value and before the semicolon.

• It does not change the specificity of the selector itself.

• Aim for better selector structure before reaching for this keyword.
