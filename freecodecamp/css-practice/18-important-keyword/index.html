The !important Keyword

Maximum priority. Final override. Use with caution.

What is !important?

The !important keyword in CSS is used to give a style rule the highest priority, allowing it to override any other declarations for a property, including inline styles.

CSS
.para {
  background-color: black !important;
  color: white !important;
}

The Specificity Killer

Normally, CSS follows specificity rules (Inline > ID > Class > Tag). However, !important breaks this hierarchy. It forces the browser to apply the style regardless of how specific other selectors are.

HTML
<p class="para-demo" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
Live Visual Output

This is a paragraph.

Note: Even though the element has lightblue background set via inline styles, the black background from our CSS (with !important) is winning.

When to use !important?

While often discouraged, !important has legitimate use cases:

  • Overriding Third-Party Styles: When using external libraries or frameworks where you can't modify the source CSS.
  • Utility Classes: Global helper classes like .hidden { display: none !important; } that should always work.
  • Quick Debugging: Temporarily forcing a style to see if specificity is the issue.

Best Practices

Overusing !important can make your CSS a nightmare to maintain. Follow these rules:

  • Always try to use higher specificity selectors first before reaching for !important.
  • Never use it for site-wide defaults or structural styling.
  • Document why you used it if it's in a production codebase.
  • Think of it as a last resort, not a standard tool.

Summary

• !important gives a property the highest possible priority.

• It can even override inline styles.

• It does not change the specificity score of the selector itself.

• Use it sparingly to keep your CSS maintainable.
