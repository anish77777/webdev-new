CSS Inheritance

Styles passed down from parent to child.

The Family Tree

Inheritance determines how styles are passed down from parent elements to their children. Just like traits in a family, certain properties flow automatically through the DOM tree.

Parent (color: amber)
Child 1 (Inherited)
Child 2 (Inherited)

Inherited vs. Non-Inherited

Not all CSS properties are passed down by default. Understanding which ones do is key to efficient styling.

✅ Inherited

  • color
  • font-family
  • line-height
  • text-align

❌ Non-Inherited

  • margin
  • padding
  • border
  • background

Basic Inheritance Demo

In this example, we set the color to blue on the parent <div>. Because color is an inherited property, the child <p> automatically adopts the blue color without any extra CSS.

HTML
<div style="color: blue;">
  Parent text (Blue)
  <p>Child text (Inherits Blue)</p>
</div>

The "inherit" Keyword

You can force any property to be inherited using the inherit keyword, even for properties that don't normally inherit (like padding).

HTML
<div style="padding: 30px;">
  <p style="padding: inherit;">
    Child inheriting 30px padding.
  </p>
</div>
Live Visual Output

Summary

• Inheritance flows from parent to child.

• Text styles (color, font) are usually inherited.

• Box model styles (margin, border) are usually not inherited.

• Use the inherit keyword to force traits down the tree.
