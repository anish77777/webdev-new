Understanding appearance: none

Take complete control over default browser UI elements.

1. What is appearance: none?

Browsers apply default styling to many elements, especially inputs. appearance: none allows you to hide these default aspects, giving you a "blank canvas" to build your own custom indicators instead.

.custom-input {
    appearance: none;
    /* Now you can style it from scratch */
}

2. Custom Checkbox Demo

By using appearance: none, we can turn a standard checkbox into a stylized component. Behind the scenes, it's still a functional checkbox, but with custom visual indicators.

.checkbox {
    appearance: none;
    width: 20px;
    height: 20px;
    border: 2px solid var(--accent-emerald);
    border-radius: 6px;
    position: relative;
    cursor: pointer;
    transition: all 0.2s cubic-bezier(0.4, 0, 0.2, 1);
}

.checkbox:checked {
    background-color: var(--accent-emerald);
}

3. WebKit Search Inputs

WebKit browsers (like Safari) add default icons and cancel buttons to type="search" inputs. appearance: none helps you remove these to maintain cross-browser consistency.

4. Benefits & Considerations

Benefits: Consistent cross-platform styling, larger tap targets for mobile, and improved accessibility (contrast).

Considerations: You must manually re-implement important features like focus rings and error indicators so they aren't lost.
