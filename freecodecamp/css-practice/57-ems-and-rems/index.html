EMS and REMS in CSS

Master modular and accessible layouts. Learn how to use relative units that scale with your typography and user preferences.

üß± The em Unit: Local Relative

The em unit is relative to the font size of the element itself. It's perfect for modular components like buttons, where you want padding and margins to scale proportionally if the font size changes.

Interact: Modular Button Scaling
Dynamic Button

Note how padding, border-radius, and shadow scale perfectly with the font size because they use em units.

Modular Component CSS

.modular-btn {
font-size: 16px;
padding: 0.8em 1.6em;
border-radius: 0.5em;
box-shadow: 0 0.5em 1em rgba(...);
}

üåê The rem Unit: Root Relative

A rem unit is relative to the font size of the root (html) element. This is preferred for typography because it respects user browser settings, making your site more accessible.

Standard root text (16px default).

I am 1.5rem (24px)

If a user scales their browser font-size to 20px, this 1.5rem element will automatically grow to 30px!

Accessible Typography CSS

.header {
font-size: 2rem; /* 32px by default */
}

.para {
font-size: 1rem; /* 16px by default */
}

üöÄ When to use what?

  • ‚ú®
    Use REM for: Font sizes and global spacing (margins/padding between sections). This ensures the whole layout scales with accessibility settings.
  • ‚ú®
    Use EM for: Sizing internal properties of a component (like the padding of a button) so it maintains its aspect ratio regardless of its font size.
