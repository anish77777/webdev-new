CSS Specificity

A fundamental concept that determines which styles are applied to an element when multiple rules could apply.

The Specificity Score

Specificity is calculated as a four-part value (a, b, c, d). Each part carries a different weight that never "rolls over" into the next.

The Hierarchy of Power

Selector types are grouped into weight categories. Below is the breakdown from highest to lowest specificity.

1.0.0.0

Inline Styles

Applied directly to the element via the style attribute. It overrides any internal or external CSS.

<p style="color: red;">High Specificity</p>
0.1.0.0

ID Selectors

Using the #id of an element. Each ID contributes 1 to the second part of the value.

#para { color: red; }
0.0.1.0

Classes, Attributes & Pseudo-classes

Includes .class, [type="text"], and :hover. These have a moderate level of specificity.

.example-para { color: green; }
0.0.0.1

Type Selectors & Pseudo-elements

Includes div, h1, ::before, and ::after. These are the lowest weight real selectors.

p { color: blue; }
0.0.0.0

Universal Selector (*)

The universal selector is a special selector that matches any and all elements in the document. It is frequently used for CSS resets (e.g., setting margin and padding to 0) to ensure a consistent starting point across different browsers.

It carries the absolute lowest specificity score. It contributes 0 to every part of the calculation, meaning any other selector type will override it.

* {
  margin: 0;
  padding: 0;
}

The Specificity Challenge

Observe how different selectors compete for the same element. Each level up the hierarchy completely overrides the ones below it.

<p id="unique" class="highlight">This text has multiple styles applied.</p>

#unique { color: purple; } /* Score: 0, 1, 0, 0 (High) */
.highlight { color: green; } /* Score: 0, 0, 1, 0 (Moderate) */
p { color: red; } /* Score: 0, 0, 0, 1 (Low) */
* { color: blue; } /* Score: 0, 0, 0, 0 (Zero) */

Result: The text will be Purple because the ID selector wins the specificity battle.

Internal vs. External CSS

Internal CSS

Defined within a <style> element in the head. It has lower specificity than inline but can override external styles if placed later or using stronger selectors.

External CSS

Linked via <link rel="stylesheet">. Provides the best maintainability for large projects despite having the same selector-based specificity as internal.

Rules in Conflict

When multiple rules apply, the one with the higher specificity score wins.

<p class="example-para" id="para">Example paragraph</p>

#para { color: red; } /* Score: 0, 1, 0, 0 */
.example-para { color: green; } /* Score: 0, 0, 1, 0 */
p { color: blue; } /* Score: 0, 0, 0, 1 */
