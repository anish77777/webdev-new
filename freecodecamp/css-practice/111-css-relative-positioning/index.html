CSS Positioning

Static vs. Relative: Moving without Breaking

01. Static Positioning

Default

By default, every element has position: static. Elements follow the normal document flow: they sit one after another, from top to bottom and left to right, according to their HTML order.

Normal Flow Demo

1
Static Box
3

The "Static Box" is right where it should be. top, left, etc. have no effect here.

p {
  position: static; /* The Default */
}

02. Relative Positioning

Offset

Relative positioning allows an element to be shifted from its original position without disrupting the rest of the layout. The space it originally occupied is preserved.

Offset Demo

1
Relative Box
3

Notice how Box 3 didn't move! The "Relative Box" is shifted 30px down and 30px right, but its "ghost" still takes up space in the flow.

.relative-box {
  position: relative;
  top: 30px;
  left: 30px;
}

Quick Comparison

Feature Static Relative
In Document Flow? Yes Yes
Accepts Offsets? No Yes (top, right, bottom, left)
