@font-face At-Rule

Defining Custom Typefaces in CSS

1. What are At-Rules?

At-rules are CSS statements providing instructions to the browser. They start with an @ symbol. Common examples include @media, @keyframes, and @font-face.

2. The @font-face Syntax

With @font-face, you can define a custom font by specifying its name and source file. This allows you to go beyond standard web-safe fonts.

@font-face {
  font-family: "MyCustomFont";
  src: url("path/to/font.woff2") format("woff2");
}

Once defined, you can use the name (e.g., "MyCustomFont") in your font-family properties throughout your stylesheet.

3. Key Descriptors

Descriptors are used inside the curly brackets to customize the font face characteristics.

The src Descriptor

This is required. it provides a list of font resources. You can include multiple formats to ensure compatibility across different browsers.

@font-face {
  font-family: "BrandFont";
  src: url("font.woff2") format("woff2"),
       url("font.woff") format("woff"),
       url("font.otf") format("opentype");
}

Browsers will check each resource in order. WOFF2 is generally preferred for its superior compression.

4. Font Formats

Different formats provide different levels of compression and feature support.

Format Name File Extension Description
WOFF / WOFF2 .woff / .woff2 Web Open Font Format. Best for web use with high compression.
OpenType .otf Scalable font format developed by Adobe and Microsoft.
TrueType .ttf Older format, widely supported on operating systems.
