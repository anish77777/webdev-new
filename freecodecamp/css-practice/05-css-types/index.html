Inline, Internal, & External CSS

Mastering the three ways to apply styles to your web pages.

The Three Approaches

CSS can be applied to a webpage in three main ways: inline, internal, or external.

Each method has its own use case, advantages, and limitations. Knowing when to use each one is essential for writing clean, efficient, and maintainable code.

Method 1 Inline CSS

Inline CSS is written directly within an HTML element using the style attribute. It applies styles to a specific element.

<p style="color: green;">This is an inline-styled paragraph.</p>

Use Case: Quick, one-off styles or overriding other styles locally.

Warning: Should be avoided in most cases as it clutters HTML and is hard to maintain.

Method 2 Internal CSS

Internal CSS is written within the <style> tags inside the <head> section. It applies styles to the entire page.

<head>
  <style>
    p { color: blue; }
  </style>
</head>

Use Case: Single-page websites or styles that don't need to be reused elsewhere.

Downside: Doesn't promote reusability across multiple pages and mixes HTML with CSS.

Method 3 External CSS

External CSS is written in a separate .css file and linked using the <link> element. This is the preferred method in professional development.

<!-- HTML -->
<link rel="stylesheet" href="styles.css">

/* styles.css */
p { color: red; }

Use Case: Large projects, consistent styling across multiple pages.

Benefits: Separation of concerns, reusability, and easier maintenance.

The Link Element

The <link> element is the bridge between your HTML and external resources.

  • rel: Specifies the relationship (e.g., stylesheet).
  • href: Specifies the URL or path to the CSS file.

Comparison Summary

Feature Inline Internal External
Scope Single Element Single Page Multiple Pages
Maintainability Low Medium High
Best For Hotfixes Single Pages Large Projects
Pro Tip: In most cases, external CSS should be your go-to approach for efficient and scalable web development.
