Type Selectors

Targeting HTML elements by their tag name.

The Foundation

Type selectors (or element selectors) target elements based on their HTML tag name. They allow you to apply global styles to every instance of an element, like styling all paragraphs or all headings at once.

styles.css
p {
  color: blue;
}

Low Specificity

SCORE: 0.0.0.1

Type selectors are efficient but have low specificity. While they are stronger than the Universal Selector, they are easily overridden by classes, IDs, and inline styles.

The Class Override

When a type selector and a class selector target the same element, the class selector wins because its specificity weight in the third column (c) is greater than the type selector's weight in the fourth column (d).

Conflict Example
p { color: blue; } /* Score: 0,0,0,1 */

.para { color: red; } /* Score: 0,0,1,0 */

I am a paragraph with the class "para". I am red because the class selector overrides the type selector.

Best Practices

• Use type selectors for base styling (like setting default body font or h1 sizes).

• Use classes when you need specifics that should override those base styles.

• Remember: Tag names are the building blocks, but classes are the design controllers.
