Mastering Link States

Using pseudo-classes to provide interactivity and feedback.

The Pseudo-class Syntax

A pseudo-class is a keyword added to a selector that specifies a special state of the selected element. The syntax looks like this, where A is the selector and :B is the pseudo-class:

A:B { property: value; }

Pseudo-classes allow you to style elements based on their state or user interaction without additional HTML markup.

Static

:link

Styles unvisited links, indicating they are clickable. Default is usually blue.

a:link { color: red; }
Unvisited Link
History

:visited

Styles links that have already been visited or clicked. Helps tracking.

a:visited { color: green; }
Visited Link
UI

:hover

Changes style when the mouse pointer is over the link. Vital feedback.

a:hover { color: green; }
Hover over me
A11y

:focus

Styles when focused (via keyboard). Essential for accessibility navigation.

a:focus { outline: 2px solid orange; }
Tab to focus me
Action

:active & The LVHFA Order

The :active state occurs precisely while the link is being clicked. To ensure styles apply correctly, the recommended order is: Link, Visited, Hover, Focus, Active. 

a:active {
  color: pink;
}
