Best Practices for Styling Text Inputs

Mastering accessibility and aesthetics for modern forms.

1. Accessible Text Inputs

When styling text inputs, accessibility is paramount. Ensure your font size is readable (at least 16px to prevent iOS auto-zoom) and that text color contrasts sufficiently with the background. 

input[type="text"] {
    font-size: 1rem;
    border: 2px solid #555;
    border-radius: 4px;
    background-color: #fff;
    color: #111;
}

2. Placeholder Styling

Placeholders are often overlooked. They must be readable even before the user starts typing. Use the ::placeholder pseudo-element to ensure sufficient contrast.

input::placeholder {
    color: #555; 
    opacity: 1; 
    font-style: italic;
}

3. Modification & Zoom

Always allow users to modify inputs. For textareas, do not remove the resize handle. Ensure elements scale correctly when the page is zoomed.

textarea {
    width: 100%;
    min-height: 120px;
    resize: both; 
    box-sizing: border-box; 
}

4. Focus Indicators

Provide a noticeable indicator when an element has focus. Avoid removing the default focus ring unless you're replacing it with a custom, clearly visible one.

input:focus {
    border-color: #1e90ff;      
    box-shadow: 0 0 0 3px rgba(30, 144, 255, 0.4); 
    outline: none;              
}

5. Error States

Visual error indicators should be distinct from focus states. Use color and descriptive text (connected via aria-describedby) to alert users.

Please enter a valid email address.

input.error {
    border-color: #d93025;
    background-color: #fff5f5;
}
