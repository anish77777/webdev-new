Hiding Content Accessibility

Best Practices for Inclusive Web Development

The Accessibility Tree

The accessibility tree is a structure used by assistive technologies, such as screen readers, to interpret and interact with web content. It represents the semantic meaning of elements. When we hide content, we must choose whether it should also be removed from this tree.

Hiding Techniques

1. display: none

Effectively hides content visually and removes it from the accessibility tree. Use this only when you want to completely remove content from everyone.

.hidden { display: none; }

2. visibility: hidden

Hides content visually but keeps it in the document flow (it still occupies space). Like display: none, it removes content from the accessibility tree.

.hidden { visibility: hidden; }

3. Visually Hidden (.sr-only)

A technique to hide content from sighted users while keeping it accessible to screen readers. This is crucial for providing extra context.

.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
}

Tip: This text is hidden from sight but yours to hear!
