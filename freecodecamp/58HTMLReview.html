HTML Prep Exam Review

Comprehensive Study Guide for Structural Web Development.

HTML Basics

Role of HTML: Hypertext Markup Language defines the structure and meaning of web content.

HTML Elements: Building blocks of a webpage, typically consisting of an opening tag, content, and a closing tag (e.g., <h1>Hello</h1>).

Void Elements: These elements do not have a closing tag because they cannot contain content (e.g., <img>, <br>, <hr>).

Common Elements

  • Headings: <h1> through <h6> define structural hierarchy.
  • Paragraphs: <p> for blocks of text.
  • Div containers: <div> is a generic container with no semantic meaning, used for grouping and styling.

Identifiers & Grouping

IDs: Unique identifiers (id="main-title"). An ID can only be used once per page.

Classes: Used to group multiple elements for shared styling (class="btn-primary").

Linking & Entities

HTML Entities: Used to display reserved characters (e.g., &lt; for < and &amp; for &).

External Files: Use <link> for CSS in the head and <script> for JavaScript.

Boilerplate & SEO

Boilerplate: The standard starting structure: <!DOCTYPE html>, <html>, <head>, and <body>.

UTF-8 Encoding: Ensures characters from all languages display correctly (<meta charset="UTF-8">).

SEO & Social Sharing

  • Meta Description: Impacts SEO by providing a page summary in search results.
  • Open Graph (OG) Tags: Enhances how links look when shared on social media.

Media & Multimedia

Replaced Elements: Content whose dimensions are determined by an external resource (e.g., <img>, <iframe>, <video>).

Multimedia Tags: Use <audio> and <video> to embed content directly. For external videos (like YouTube), use <iframe>.

SVGs: Scalable Vector Graphics are XML-based images that stay sharp at any zoom level and are perfect for icons/logos.

Paths & Link Behavior

Target Attribute: Controls where a link opens (e.g., target="_blank" for a new tab).

Path Syntax

  • Absolute Path: Full URL (https://example.com/image.png).
  • Relative Path: Navigates based on current directory.
  • /: Root directory.
  • ./: Current directory.
  • ../: Up one directory level.

Internal Links: Use href="#section-id" to jump to an element with that specific ID on the same page.

Semantic HTML

Semantic elements give meaning to the structure rather than just defining appearance.

Structural Elements

  • <header> & <footer>: Top and bottom of sections/documents.
  • <nav>: Primary navigation links.
  • <main>: The unique, principal content of the body.
  • <article>: Self-contained, independent content.
  • <section>: Groups of related content.

Text-Level Semantics

<em>Stress emphasis (usually italic)</em>
<strong>Strong importance (usually bold)</strong>
<blockquote>External source quotation</blockquote>
<abbr title="Markup Language">ML</abbr>
<time datetime="2024-05-15">May 15</time>

Forms & User Input

Forms gather user data using <form method="POST" action="/submit">.

Input Types & Attributes

Common types: text, email, password, checkbox, radio, number, date, submit.

<input 
  type="text" 
  name="user_name" 
  placeholder="Enter name"
  required 
  minlength="3"
>

Labels & Grouping

  • Labels: Use <label for="id"> to associate text with an input.
  • Fieldset & Legend: Use for logical grouping of related inputs (like a set of radio buttons).

Table Elements

Tables are structured using rows (<tr>), headers (<th>), and data (<td>).

<table>
  <caption>Student Grades</caption>
  <thead>
    <tr><th>Name</th><th>Grade</th></tr>
  </thead>
  <tbody>
    <tr><td>Alex</td><td>A</td></tr>
  </tbody>
  <tfoot>
    <tr><td colspan="2">Average: B+</td></tr>
  </tfoot>
</table>

Accessibility (POUR)

WCAG Principles: Perceivable, Operable, Understandable, Robust.

ARIA: Accessible Rich Internet Applications (e.g., role="alert", aria-label).

Key Best Practices

  • Focus Management: Use tabindex="0" or -1. Never use > 0.
  • Alt Text: Always provide text alternatives for images.
  • Link Text: Use descriptive text ("Read our privacy policy") instead of "Click here".

Essential Development Tools

  • HTML Validator: Checks syntax for errors.
  • DOM Inspector: Visualizes the document tree in real-time.
  • DevTools: Browser-integrated debugging suite (F12).
