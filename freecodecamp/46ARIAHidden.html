What Is the aria-hidden Attribute, and How Does It Work?

If you ever need to display content while at the same time hiding it from people who use assistive technology, like screen readers, you can use the aria-hidden attribute.

You just need to add it to the HTML element that you want to hide and set its value to true, like this: aria-hidden="true".

How it works

This attribute hides the element and all of its children from the accessibility tree, while keeping them visible on the page.

Common use cases include:

  • Icons and images that only have a decorative purpose.
  • Duplicated content.

IMPORTANT: aria-hidden only hides content from assistive technology. If content should be hidden from everyone, use display: none or visibility: hidden in your CSS instead. For example, a collapsed hamburger menu should be hidden from everyone.

Example: Hiding a Decorative Icon

In this example, we hide an icon from the accessibility tree by adding the aria-hidden="true" attribute. We keep the text exposed so assistive technologies don't hear redundant information.

<button> <i class="fa-solid fa-gear" aria-hidden="true"></i> <span class="label">Settings</span> </button>

When you DON'T need aria-hidden

You do not need to use aria-hidden when:

  • The HTML element already has a hidden attribute.
  • The element/ancestor is hidden with display: none.
  • The element/ancestor is hidden with visibility: hidden.

In these cases, the element is already removed from the DOM or hidden from the accessibility tree automatically.

Best Practices

  • Focusable Elements: Never use aria-hidden on an element that can receive keyboard focus (like a link or button). This causes screen reader users to focus on "nothing".
  • Inheritance: Setting aria-hidden="false" on a child won't work if a parent already has aria-hidden="true".
  • Sparingly: Content and functionality should generally be available to everyone. Use aria-hidden primarily to clean up decorative or duplicate information.

By following these best practices and testing with assistive technologies, you can create more inclusive online experiences for everyone.
