Link Text Accessibility Guide

Learn why descriptive link text serves users better and see examples of how to write it effectively.

The Benefits of Good Link Text

The first visible benefit of a good link text is that it makes it easier for everyone to find information quickly. Descriptive links help users know where they're headed and what they'll access. This ensures the user doesn't feel lost and improves the overall user experience.

Screen Reader Users

For those using screen readers, a clear and descriptive link text is a must. Screen readers read the link text aloud, so a text like "Read our accessibility guide" is way better than "Click here."

Cognitive Accessibility

Making link text clear and descriptive isn't just beneficial for those living with visual impairments. Descriptive links also help people with cognitive disabilities by providing clear context.

Best Practices Checklist

Here are key rules to follow when writing link text:

  • Visually Distinct: Use underlining or other visual cues so users can easily identify links.
  • Avoid Generics: Words like "here", "click here", and "more-info" provide zero context.
  • Be Concise: Aim for 2-5 words that convey the purpose.
  • No Jargon: Avoid technical terms or abbreviations users might not know.
  • Destination Focused: Describe where the link goes (e.g., "user behavior results") rather than the action ("click here").
  • Unique: Donâ€™t use the same text for links that go to different places.
  • Contextual: Integrate links naturally into sentences (e.g., "visit our events page" instead of "Click here for events").

Examples: Good vs. Bad

Let's look at specific use cases.

Example 1: Event Details

Bad Practice

Code:

<a href="webinar-details-link">Details</a>

Why it fails: "Details" is too vague. It doesn't tell the user what details they are getting.

Best Practice

Code:

<a href="webinar-details-link">
  Get details about our upcoming webinar
</a>

Why it works: It gives specific context ("upcoming webinar"), helping the user decide if they want to click.

Example 2: Blog Post

Bad Practice

Code:

<a href="/blog-post-link">Read more</a>

Why it fails: "Read more" is meaningless out of context (e.g., in a screen reader's list of links).

Best Practice

Code:

<a href="/blog-post-link">
   Read our latest blog post on web accessibility
</a>

Why it works: It sets clear expectations about the content ("latest blog post on web accessibility").

Example 3: Topic Info

Bad Practice

Code:

<a href="/link-to-topic">More info</a>

Why it fails: Vague and unhelpful for non-linear navigation.

Best Practice

Code:

<a href="/link-to-topic">
   Learn more about our accessibility efforts
</a>

Why it works: clearly describes the topic ("accessibility efforts").
